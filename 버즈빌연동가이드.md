# 버즈빌 API 연동 가이드

## 명세서 수령 후 확인할 항목

1. **API 엔드포인트 URL**
2. **인증 방식** (API Key, Bearer Token 등)
3. **필수 파라미터** (user_id, mission_type 등)
4. **요청 형식** (JSON, Form 등)
5. **응답 형식 및 성공/실패 코드**

---

## 수정할 파일

### `apps-script/Code.gs`

```javascript
// 1. 상단 설정값 수정
const BUZZVIL_API_URL = '실제_API_URL';
const BUZZVIL_API_KEY = '실제_API_KEY';

// 2. callBuzzvilApi 함수 수정 (명세에 맞게)
function callBuzzvilApi(data) {
  const payload = {
    // 명세서에 맞게 필드 구성
    user_id: data.phone,  // 또는 다른 식별자
    // ... 추가 필드
  };

  const options = {
    method: 'POST',
    contentType: 'application/json',
    headers: {
      'Authorization': 'Bearer ' + BUZZVIL_API_KEY,
      // 명세서에 따라 헤더 추가
    },
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  };

  const response = UrlFetchApp.fetch(BUZZVIL_API_URL, options);
  return JSON.parse(response.getContentText());
}

// 3. doPost 함수에서 주석 해제
function doPost(e) {
  try {
    const data = JSON.parse(e.postData.contents);
    saveToSheet(data);

    // 아래 주석 해제
    const buzzvilResult = callBuzzvilApi(data);

    // 버즈빌 응답을 시트에 기록 (선택)
    // updateBuzzvilResponse(sheet.getLastRow(), buzzvilResult);

    return createResponse({ success: true });
  } catch (error) {
    return createResponse({ success: false, error: error.message });
  }
}
```

## 작업 순서

1. [ ] 명세서 확인 및 파라미터 정리
2. [ ] `BUZZVIL_API_URL` 설정
3. [ ] `BUZZVIL_API_KEY` 설정
4. [ ] `callBuzzvilApi` 함수 payload 수정
5. [ ] `doPost`에서 `callBuzzvilApi` 주석 해제
6. [ ] Apps Script 저장
7. [ ] **새 배포** (배포 > 배포 관리 > 새 배포)
8. [ ] 테스트 폼 제출로 연동 확인

## 테스트 방법

1. 랜딩페이지에서 테스트 폼 제출
2. 구글시트에 데이터 저장 확인
3. 버즈빌 대시보드에서 미션 성공 기록 확인
4. (선택) 시트에 버즈빌 응답 기록 확인

---

## 주의사항

- 코드 수정 후 반드시 **새 배포** 필요
- API Key는 코드에 직접 노출되므로 공개 저장소 주의
- 버즈빌 API 호출 실패해도 시트 저장은 완료되도록 구현됨
