# 랜딩페이지 시스템 차별점

## 인프라
- **서버 비용 0원** — Cloudflare 무료 티어로 전부 운영 (Pages + Worker + D1)
- **하루 10만 건 처리** — 무료인데 트래픽 걱정 없음. 유료 전환 없이도 웬만한 규모 커버
- **글로벌 CDN** — Cloudflare 엣지 서버에서 서빙. 한국 어디서든 빠름
- **SSL 자동 적용** — HTTPS 무료, 설정 필요 없음

## 운영 편의
- **랜딩 추가 5분** — 폴더 복사 + git push면 끝. 개발자 없이도 가능
- **하나의 백엔드로 모든 랜딩 처리** — 랜딩이 100개여도 Worker 1개로 운영
- **커스텀 도메인 무제한** — 업체별 자체 도메인 연결 가능
- **구글시트 실시간 확인** — 광고주가 익숙한 구글시트에서 바로 DB 확인. 별도 대시보드 불필요

## 유연성
- **랜딩별 폼 필드 자유 설정** — 보험은 이름/연락처/생년월일, 부동산은 이름/연락처/관심지역. 코드 수정 없이 script.js에서 fields만 바꾸면 됨
- **랜딩별 다른 구글시트 연결 가능** — 광고주마다 자기 시트로 데이터 수신
- **시트 헤더 자동 생성** — 빈 시트에 데이터 들어오면 헤더가 알아서 생김

## 데이터 보호
- **동일 번호 중복 제출 차단** — 랜딩별로 D1에서 중복 체크. 허위 DB 방지
- **프론트 + 백엔드 이중 차단** — 로컬스토리지 1차 + D1 2차
- **D1 원본 DB 보관** — 광고주가 시트를 실수로 수정해도 원본 안전
- **구글시트 저장 우선** — 시트 저장 실패하면 사용자에게 에러. 데이터 유실 없음

## 보안
- **CORS 제한** — 허용된 도메인에서만 폼 제출 가능. 외부 스팸 차단
- **Rate Limiting** — IP당 1분 5회 제한. 봇 대량 제출 방어
- **에러 자동 로깅** — 장애 발생 시 D1에 기록. 원인 추적 가능

## 향후 확장성
- **Webhook 알림** — 폼 제출 시 슬랙/텔레그램으로 실시간 알림
- **UTM 파라미터 자동 수집** — 광고 유입 경로(utm_source, utm_medium 등)를 자동으로 시트에 같이 저장
- **A/B 테스트 지원** — 같은 랜딩의 A/B 버전 트래픽 분배 + 전환율 비교

## 일반 외주 업체와 비교

|---|---|---|
